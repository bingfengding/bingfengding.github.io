用hexo+github搭建自己的博客并使用next设置主题

发表于 2018-04-09 | 分类于 前端 ， 其他 | | 阅读次数:
字数统计: 3,480 字 | 阅读时长 ≈ 13 分钟

闲谈

早就想写搭建1个博客，但总是没什么时间，最近比较空闲，抽空搭建并写下来方便记忆。
要说博客的用处，最大的还是将自己需要的资料进行记录，之后有需要就可以直接翻看，而不是各种度娘查资料。
首先度娘了不少，发现发现hexo+github蛮不错的，搭建之后在绑定到自己的域名下，添加上各种功能，感觉萌萌哒。结果搭建过程中，遇到了不少坑。把自己的坑也说下。免得后来人与我一样。
那么开始吧。

环境

git node.js安装

当然开始之前需要基本的环境，git node.js 基本上准备搭建博客的应该都有了吧。如果没有，去对应的官网下载，安装很简单，都是一路next就好。安装完毕建议去cmd运行
git –version
node -v
npm -v
进行验证是否已经成功安装。

github

既然文章名为hexo + github 。那么github的注册也是必须的。

申请github

点击-> GitHub 进入官网注册帐号
注册这方面不是什么问题，就不详细说明了，实在不会，使用谷歌浏览器，右键选择翻译全文。再不会。。就放弃吧。

创建版本库

进入Repositories然后点击new

Repository name 的前缀与你的账号相同，后面跟上 github.io 不要在意我的报错提示，因为我已经生成了所以会提示存在。其他选项默认就好，设置好之后点击create repository生成。
当你申请完毕之后，你进入 username.github.io。(username表示你的注册名)你会发现已经有1个网站出现了。这是github默认的。只要这么创建就会自动将其作为github的博客。

安装hexo

接下来，就是安装hexo，这是1个基于node.js的静态博客，官网也是搭建在GitHub上

  • 新建一个blog文件夹,该文件夹用于存放你的博客文件,(位置随意，你自己选喜欢的)然后右键单击选择“Git Bash”(如果正常安装了git，那么你的右键肯定会有这2个选项的) 通过npm安装hexo
    $ npm install -g hexo-cli 由于我已经安装过了，就不重复了，直接回车就好。
    安装完毕再次安装
    $ npm install hexo –save
    都安装完成之后，严重会是否正确
    $ hexo -v
    运行hexo
    初始化hexo
    $ hexo init 安装生成器
    $ npm install
    运行hexo,以后要在本地运行博客只要输入该命令即可
    $ hexo s -g 打开浏览器，输入localhost:4000,就可以在本地看到你的个人博客了 由于自己已经更新过了，所以就网上搜索了个原本安装之后的截图效果。
    停止运行就是按ctrl+C。
    在这里先说明几个常用的命令
    hexo clean 这个是清除hexo缓存。如果之后进行数据上传github的时候一定要先清除下。否则因为缓存原因导致新数据没上传。
    hexo deloy 可以简写 hexo d 也就是上传到你的指定网站内。通过这文章搭建也就是上传到github上。
    hexo config 进行配置设置 如果端口号4000 被占用可以通过这里修改
    hexo generate 可以简写 hexo g 生成静态页面
    hexo new 文章名 生成1个新的文章
    hexo server 简写 hexo s 在本地环境下生成
    以及多个一起可以写成 hexo s -g 也可以 hexo g -s
    了解大致命令之后接下来会针对这些进行介绍

    管理博客

    进行数据配置
    通过编辑器打开blog/_config.yml文件，进行配置 我个人使用的是phpstorm，属于很笨重的编辑器，如果只是想单纯搭建博客，可以使用Sublime_Text编辑器。如果你连这个都不想安装，可以使用windows自带的文本文档打开 #博客名称

    #副标题

    language: zh-CN

    permalink_defaults:
    source_dir: source
    public_dir: public
    tag_dir: tags
    archive_dir: archives
    category_dir: categories
    code_dir: downloads/code
    i18n_dir: :lang
    skip_render:
    new_post_name: :title.md # File name of new posts
    default_layout: post
    titlecase: false # Transform title into titlecase
    external_link: true # Open external links in new tab
    filename_case: 0
    render_drafts: false
    post_asset_folder: false
    relative_link: false
    future: true
    highlight:
    enable: true
    line_number: true
    auto_detect: true
    tab_replace:
    default_category: uncategorized
    category_map:
    tag_map:
    #日期格式
    date_format: YYYY-MM-DD
    time_format: HH:mm:ss
    #分页，每页文章数量
    per_page: 10
    pagination_dir: page
    #博客主题
    theme: next
    #发布设置
    deploy:
    type: git
    #bingfengding改为你的github用户名
    repository: git@github.com:bingfengding/bingfengding.github.io.git
    branch: master
    写一篇文章
    输入创建文章命令，生成一个md文件(/blog/source/_posts/)
    $ hexo new “helloWorld”
    用编辑器打开hello.md文件,编写之后就保存

    发布博客

    设置git身份信息
    $ git config –global user.name “你的用户名”
    $ git config –global user.email “你的邮箱” 安装hexo git插件
    $ npm install hexo-deployer-git –save
    发布更新博客
    $ hexo d -g
    发布成功后，访问yourname.github.io看下成果

安装主题next

依旧是打开你的右键git base here 打开你的blog文件夹
git clone https://github.com/iissnan/hexo-theme-next themes/next

这原理很简单，就是将主题文件拷贝至站点目录的 themes 目录下， 然后修改下配置文件即可。

启用主题

打开blog/_config.yml， 找到 theme 字段，并将其值更改为 next。

NexT 主题安装完成。下一步我们将验证主题是否正确启用。在切换主题之后、验证之前， 我们最好使用 hexo clean 来清除 Hexo 的缓存。

验证主题

hexo s –debug
加上debug是为了方便检测异常问题，不加也没事。打开之后检查是否正常
当你看到站点的外观与下图所示类似时即说明你已成功安装 NexT 主题。这是 NexT 默认的 Scheme —— Muse

设置头像

编辑 主题配置文件， 修改字段 avatar， 值设置成头像的链接地址。其中，头像的链接地址可以是完整的url或者是站点内的相对地址，放置在 source/images/ 目录下 配置为：avatar: /images/avatar.png

应用名随意

安全设置

因为AppID以及AppKey是暴露在外的，因此如果一些别用用心之人知道了之后用于其它目的是得不偿失的，为了确保只用于我们自己的博客，建议开启Web安全选项，这样就只能通过我们自己的域名才有权访问后台的数据了，可以进一步提升安全性。
选择应用的设置的安全中心选项卡:

为博客更改域名

对于有自己域名的人，肯定不会满足于使用github.io 这个现实为博客名的。因此这里也介绍下如何绑定自己的域名下。以自己的万网介绍。
进入后台管理。

点击解析，进入如下页面进行点击 添加解析 ：

最后还有一步，在你的本地博客 source 目录下，新建一个文件，命名为 CNAME ，内容为你设置的域名

本来之前是用的ip，绑定的A记录，后来登录github推荐使用CNAME记录，因为ip有可能受到其他影响变化等。依照这样配置创建完毕，10分钟之后，你的自己域名的博客就产生了。
